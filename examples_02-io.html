Saving and loading ScenesÂ¶

The native file structure of kite is based on NumPy binary files together with YAML configuration files holding the necessary information to fully reconstruct instances of

  • Quadtree,
  • Covariance,
  • and Meta,

through serializing the correspondig Config classes (Module Reference).

Note

The expensive calculation of kite.Covariance.covariance_matrix is saved in the YAML file!

Importing data from a foreign file format and transfering it to kite’s native format is as easy as 1, 2, 3...

from kite import Scene
sc = Scene.import_data('unwrap_ll.grd')  # GMT5SAR import
sc.save('kite_scene')

The kite file structure consists of only two files:

project_name.yml
project_name.npz
